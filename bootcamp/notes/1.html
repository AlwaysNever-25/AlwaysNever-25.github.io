Semantic HTML

Woof, boy. This is big. So.... semantic. Basically, it doesn't use div tags with classes and all at the highest level of a website. They use their own tags instead so that div use are generally not recommended anymore, at least not in the highest level. Div could still be used inside of the new semantic HTML tags.

What's the benefit? Readable codes, by user that doesn't understand programming or has disabilities, other programmer that wants to read your code, and crawler that reads your website (for SEO).

Examples? Navbar now uses nav instead of class="navbar". Main now uses main, and there can only be one main tag at a time.
